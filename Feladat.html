Nagy Beadandó

A feladatokban kért algebrai adatszerkezetekhez hozzá kell rendelni az Eq és Show típusosztályokat, amennyiben a leírás nem kér mást. A megfelelő tesztelés érdekében kérlek, az alábbi függvényeket másold be a kódodba:

showState a = show a
showMage a = show a
eqMage a b =  a == b
showUnit a = show a
showOneVOne a = show a

Típus magyarázó

type Name = String
type Health = Integer
type Spell = (Integer -> Integer)
type Army = [Unit]
type EnemyArmy = Army
type Amount = Integer

Első feladat

a)

Alkosd meg a State paraméteres algebrai adatszerkezetet. Ez tartalmaz két konstruktort Alive és Dead. Az Alive rendelkezzen a típus paraméterrel, a Dead csak önmagában áll.

Kiírásnál (Show) ne jelenjen meg az Alive konstruktor.

data State = ...

Tesztesetek:

showState (Alive 5) == "5"
showState (Alive True) == "True"
showState Dead == "Dead"
showState (Alive 'a') == "'a'"

b)

Alkosd meg az Entity algebrai adatszerkezetet. Az adatszerkezet a Golem és HaskellElemental egységeket fogja ábrázolni, amiben a Health van egyedül eltárolva.

data Entity = ...

c)

A Mage adatszerkezet tároljon el a Master konstruktorban egy mágust. A mágus rendelkezik névvel (String), életerővel (Health) és egy varázslattal, amit a csatában használ (Spell).

A Mage-nek kiírásnál (Show) csak a neve látszódik. Két mágust a neve és életereje alapján tudsz megkülönböztetni (Eq).

data Mage = ...

Tesztesetek:

showMage papi == "Papi"
showMage java == "Java"
showMage (Master "Skver" 4 id) == "Wounded Skver"
showMage (Master "Papi" 4 id) == "Wounded Papi"

eqMage papi papi
not $ eqMage papi traktor
eqMage jani jani
not $ eqMage java jani

Az alábbi mágusok a további tesztekben megjelennek.

papi = let 
    tunderpor enemyHP
        | enemyHP < 8 = 0
        | even enemyHP = div (enemyHP * 3) 4
        | otherwise = enemyHP - 3
    in Master "Papi" 126 tunderpor
java = Master "Java" 100 (\x ->  x - (mod x 9))
traktor = Master "Traktor" 20 (\x -> div (x + 10) ((mod x 4) + 1))
jani = Master "Jani" 100 (\x -> x - div x 4)
skver = Master "Skver" 100 (\x -> div (x+4) 2)
potionMaster = 
  let plx x
        | x > 85  = x - plx (div x 2)
        | x == 60 = 31
        | x >= 51 = 1 + mod x 30
        | otherwise = x - 7 
  in Master "PotionMaster" 170 plx

d)

Alkosd meg a Unit szerkezetet. Ez egy M vagy E konstruktorral tárol el egy State-be csomagolt Mage-et vagy Entity-t.

Kiíráskor (Show) a konstruktor ne jelenjen meg.

data Unit = ...

Tesztesetek:

showUnit (M (Alive papi)) == showMage papi
showUnit (E (Alive (Golem 5))) == show (Golem 5)
showUnit (E (Alive (HaskellElemental 5))) == show (HaskellElemental 5)
showUnit (E Dead) == "Dead"
showUnit (M Dead) == "Dead"

Negyedik feladat

A fight függvény segítségével megkapod az első paraméterben kapott sereg támadása után a második paraméterben megtámadott sereg állapotát.

  • Golem a vele azonos pozícióban lévő ellenségbe sebez 1 pontot
  • HaskellElemental a vele azonos pozícióban lévő ellenségbe sebez 3 pontot
  • Mage a vele azonos és hátrébb lévő ellenségre használja a képességét

Ha egy egység életereje 1 alá csökken, elesik.

fight :: EnemyArmy -> Army -> Army

Ötödik feladat

A haskellBlast függvény megkeresi, hogy egy 5 hosszú területen hol tudja a legnagyobb sebzést okozni úgy, hogy azon a területen minden egységbe 5-öt sebez. Miután megtalálta, sebzi is az egységeket azon a területen. Amennyiben egy egység életereje 1 alá csökken, az elseik.

haskellBlast :: Army -> Army

haskellBlast [] == []
haskellBlast [E (Alive (Golem 10))] == [E (Alive (Golem 5))]

haskellBlast ([M (Alive traktor)] ++ [E (Alive (Golem n)) | n <- [5,5,5,5]]) == [M (Alive (Master "Traktor" 15 (\x -> div (x + 10) ((mod x 4) + 1)))),E Dead,E Dead,E Dead,E Dead]

haskellBlast ([E (Alive (Golem n)) | n <- [5,5,5,5,5]] ++ [M (Alive papi)]) == [E Dead,E Dead,E Dead,E Dead,E Dead,M (Alive papi)]

Hatodik feladat

A multiHeal függvény vesz egy megadott mennyiségű életerőt, majd azt egyenletesen balról jobbra szétosztja az egységek között. Ha nem lehet mindenkinek ugyanannyit osztani, akkor a maradékot balról jobbra haladva osztja el, amíg tudja. Elesett egységeket nem tudsz gyógyítani.

multiHeal :: Health -> Army -> Army

Tesztesetek:

multiHeal 5 [] == []
multiHeal 5 [E Dead] == [E Dead]
multiHeal 5 [E (Alive (Golem 5))] == [E (Alive (Golem 10))] 
multiHeal 5 [E (Alive (Golem 1)), M Dead, E (Alive (Golem 1))] == [E (Alive (Golem 4)),M Dead,E (Alive (Golem 3))]
multiHeal 5 [E (Alive (Golem n)) | n <- [1,1,1,1]] == [E (Alive (Golem 3)),E (Alive (Golem 2)),E (Alive (Golem 2)),E (Alive (Golem 2))]
multiHeal 5 [E (Alive (Golem n)) | n <- [1,1,1,1,1]] == [E (Alive (Golem 2)),E (Alive (Golem 2)),E (Alive (Golem 2)),E (Alive (Golem 2)),E (Alive (Golem 2))]
multiHeal 5 [E (Alive (Golem n)) | n <- [1,1,1,1,1,1]] == [E (Alive (Golem 2)),E (Alive (Golem 2)),E (Alive (Golem 2)),E (Alive (Golem 2)),E (Alive (Golem 2)),E (Alive (Golem 1))]

Eddig voltak kötelezőek a feladatok, ezen a ponton túl lehet pontokat szerezni, LEGFELJEBB 5 pont szerezhető (pontokból több van, így lehet válogatni).

Hetedik feladat (2 Pont)

A battle függvénnyel szimulálj le egy csatát, majd add vissza a győztes csapat állapotát Just-ba csomagolva. Ha nincs győztes, adj vissza Nothing-ot.

Csata menete:

  • a két csapat megütközik
  • a egy haskellBlast becsapódik a seregedbe
  • egyenletesen gyógyítalsz 20 pontot a seregedben
  • mindkét sereg rendezi sorait és a halottakat kivonja a csata közepéről

Ez addig ismétlődik, amíg egy csapat összes tagja el nem esik.

battle :: Army -> EnemyArmy -> Maybe Army {- vagy EnemyArmy lesz az eredmény. -}

Tesztesetek:

battle [] [] == Nothing
battle [] [E (Alive (HaskellElemental 5))] == Just [E (Alive (HaskellElemental 5))]
battle [E (Alive (Golem 5))] [] == Just [E (Alive (Golem 5))]
battle [E (Alive (Golem n)) | n <- [5,5,5,5]] [E (Alive (HaskellElemental 5))] == Just [E (Alive (HaskellElemental 4))]
battle [E (Alive (Golem n)) | n <- [30,30]] [E (Alive (HaskellElemental 5))] == Just [E (Alive (Golem 40)),E (Alive (Golem 55))]
battle [E (Alive (Golem n)) | n <- [30,30]] [M (Alive potionMaster), E (Alive (HaskellElemental 5))] == Just [E (Alive (Golem 52)),E Dead]
battle [E (Alive (Golem n)) | n <- [30,30]] ([M (Alive potionMaster)] ++ [E (Alive (HaskellElemental 10)) | n <- [50,50,50]]) == Just [E (Alive (Golem 352)),E Dead] 
battle [M (Alive papi)]  [E (Alive (HaskellElemental 10)) | n <- [100..200]] == Just [M (Alive (Master "Papi" 150 id))]
battle [E (Alive (Golem n)) | n <- [50,50,50,50]] [E (Alive (HaskellElemental 10)) | n <- [50,50,50,50,50,50,50,50]] == Just [E (Alive (HaskellElemental 5)),E (Alive (HaskellElemental 5)),E (Alive (HaskellElemental 5)),E (Alive (HaskellElemental 5)),E Dead,E Dead,E Dead,E Dead]
battle [E (Alive (Golem 20)), M (Alive traktor)]  [E (Alive (HaskellElemental 10)) | n <- [100..200]] == Just [E (Alive (Golem 1840)),M (Alive (Master "Traktor" 1867 id))]

Nyolcadik feladat (2 Pont)

A chain függvény végig megy felváltva a két sereg egységein. Egy megadott mennyiséggel gyógyítja a te embereid és sebzi az ellenségeset. Minden ugrással ez az érték egygyel csökken, amíg el nem éri a nullát vagy nem tud tovább ugrani.

chain :: Amount -> (Army, EnemyArmy) -> (Army, EnemyArmy)

Tesztesetek:

chain 5 ([],[]) == ([],[])

chain 5 ([E (Alive (Golem n)) | n <- [5,5,5,5]], [E (Alive (HaskellElemental n)) | n <- [10,10,10,10]]) == ([E (Alive (Golem 10)),E (Alive (Golem 8)),E (Alive (Golem 6)),E (Alive (Golem 5))],[E (Alive (HaskellElemental 6)),E (Alive (HaskellElemental 8)),E (Alive (HaskellElemental 10)),E (Alive (HaskellElemental 10))])

Tizedik feladat (1 Pont)

Az alábbi rekurzív algebrai adatszerkezet egy párbajt ábrázol.

data OneVOne = Winner String | You Health OneVOne | HaskellMage Health OneVOne deriving Eq

A konstruktorok jelölik, hogy éppen kinek a köre jön és mennyi életerővel rendelkezik. A rekurzív szerkezet egy Winner konstruktottal fejeződik be, ami a párbaj győztesét tartalmazza.

Alkosd meg hozzá a Show instance-t a következő feltételekkel:

  • A párbaj <> jelek között jelenjen meg
  • A párnaj részei ;-vel legyenek elválasztva
  • A párbaj győztese || jelek között jelenjen meg

Tesztesetek:

showOneVOne (Winner "") == "<|| Winner  ||>"
showOneVOne (You 5 (Winner "")) == "<You 5; || Winner  ||>"
showOneVOne (HaskellMage 10 (You 5 (Winner "a"))) == "<HaskellMage 10; You 5; || Winner a ||>"
showOneVOne (You 10 (You 5 (Winner "a"))) == "<You 10; You 5; || Winner a ||>"

Tizenegyedik feladat (2 Pont)

Alkosd meg a finalBattle függvényt, ami egy párharcot fog generálni. A párharcban az felek felváltva következnek és az alábbi stratégiákat alkalmazzák:

Haskell mágus:

  • Ha az életereje kevesebb, mint 4, megfelezi ellenfele életerejét és a sajátját a négyszeresére növeli
  • Ha az ellenfele életereje 20 nál több, akkor az ellenfele életének háromszorosának negyedére csökkenti
  • Különben tizenegyet sebez ellenfelébe

Te:

  • Ha a te életerőd négynél kevesebb, a négyszeresére növeled azt
  • Ha ellenfeled életereje tizenötnél nagyobb, akkor azt a háromszorosának ötödére csökkented
  • Kölönben kilencet sebzel bele
finalBattle :: Health -> Health -> OneVOne

Teszteset:

show (finalBattle 500 500) == "<HaskellMage 500; You 375; HaskellMage 300; You 281; HaskellMage 180; You 210; HaskellMage 108; You 157; HaskellMage 64; You 117; HaskellMage 38; You 87; HaskellMage 22; You 65; HaskellMage 13; You 48; HaskellMage 4; You 36; HaskellMage 0; || Winner You ||>"
